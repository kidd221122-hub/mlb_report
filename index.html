指標分類

A. 結果型指標（Results）— 反映「發生了什麼」

  • ERA：直觀、反映真實結果，但受防守影響大、明年相關性僅 0.38
  • WHIP：穩定、易理解，但不區分安打種類

B. 技能型指標（Skills）— 預測「接下來會發生什麼」⭐ 推薦使用

  • xFIP：校正全壘打運氣的 FIP，明年相關性 ~0.52 ⭐最佳
  • SIERA：最全面的技能指標，含 K/BB/HBP/GB/FB/線性加權
  • FIP：(13×HR + 3×(BB+HBP) - 2×K)/IP + 常數，簡單有效
  • DRA-：含場地、對手、主客場調整的期望防禦率

C. 進階 Statcast 指標

  • K/9：每九局三振數 — 高，投手最能控制的技能
  • K-BB%：三振率減保送率 — 非常高，綜合控球與壓制
  • Barrel%：最佳擊球品質比例 — 中高
  • GB%/FB%：滾地球/飛球比例 — 中，影響全壘打率

三、勝負預測公式（Moneyline）

方法一：Poisson 分佈法（簡潔高效）

第 1 步：計算主隊預期得分 λ_home λ_home = 基礎得分 × 攻擊調整 × 先發調整 × 場地因子 × 天氣因子 + 牛棚調整 + 主場優勢 其中： 基礎得分 = 聯盟平均得分（約 4.5 分） 攻擊調整 = 球隊 xwOBA / 聯盟平均 xwOBA 先發調整 = 聯盟平均 xFIP / 先發投手 xFIP 場地因子 = 該球場 Run Factor（baseballsavant 可查） 天氣因子 = 1 + (溫度−60)×0.002 + 風向調整(−0.05 ~ +0.10) 牛棚調整 = 0.1 × (牛棚 xFIP − 聯盟平均 xFIP) 主場優勢 = +0.10 第 2 步：同樣計算 λ_away（無主場優勢） 第 3 步：Poisson 分佈計算勝率 P(主隊得 r 分) = e^(-λ_home) × λ_home^r / r! P(客隊得 r 分) = e^(-λ_away) × λ_away^r / r! 主隊勝率 = Σ(i>j) P(主隊=i分) × P(客隊=j分)

方法二：Monte Carlo 模擬法（Ballpark Pal 方式，更精確）

第 1 步：建立每個打者 vs 投手的對決機率分佈 每個 PA 有 15+ 種可能結果：K、BB、HBP、1B、2B、3B、HR、GO、FO、BF... 機率來自 100+ 特徵的機器學習模型（含 exit velocity、launch angle、count、history...） 第 2 步：逐局模擬 從 1 上半開始 → 逐個 PA 隨機抽樣 → 更新 base-out state → 計分 跑者進壘機率（speed）、盜壘機率、牽制機率也都隨機抽樣 第 3 步：重複 3,000–10,000 次 每次模擬產生一場完整的比賽結果 第 4 步：統計 主隊勝率 = 主隊勝的模擬次數 / 總模擬次數 預期比分 = 平均模擬得分 大小分概率 = P(總分 > N)

四、比分預測公式（Total Runs / Over-Under）

總預期得分 = λ_home + λ_away P(大分 > N) = 1 − PoissonCDF(N, 總預期得分) P(小分 ≤ N) = PoissonCDF(N, 總預期得分) 其中 PoissonCDF(N, λ) = Σ(k=0 to N) [e^(-λ) × λ^k / k!]

範例計算

假設：λ_home = 4.8（主隊 xwOBA 高於聯盟、先發 xFIP 低、場地有利）、λ_away = 3.9（客隊一般、先發稍弱）

總預期 = 8.7

P(大 8.5) = 1 − PoissonCDF(8, 8.7) ≈ 0.52

P(小 8.5) = PoissonCDF(8, 8.7) ≈ 0.48

→ 大分 8.5 有微弱優勢

六、實作流程圖

📥 資料輸入 ├─ 先發投手確認（xFIP/SIERA/K-BB% vs 對方打線） ├─ 球隊近期攻擊（xwOBA/Barrel% vs 對方投手側） ├─ 牛棚狀態（xFIP/近3天使用量） ├─ 場地因子（Run Factor + HR Factor） ├─ 天氣預報（溫度/風向/濕度/降雨） └─ 情境（主場/旅行/休息/名單） ▼ 📐 特徵工程 ├─ 計算 λ_home 和 λ_away ├─ 調整左右投配對 └─ 加入牛棚疲勞懲罰 ▼ 🎲 預測引擎 ├─ [簡易版] Poisson 分佈 → 勝率 + 大小分 └─ [進階版] Monte Carlo 3000 次模擬 → 完整分佈 ▼ 📊 輸出 ├─ 主隊勝率（如 58.7%） ├─ 客隊勝率（如 34.3%） ├─ 預期比分（如 5–3） ├─ 大分 N.5 概率 └─ 公平賠率（Fair Odds） ▼ ⚖️ 市場比較 ├─ 公平賠率 vs 運動彩券賠率 └─ EV > 0 → 下注建議

七、Python 實作範例

import math
from scipy.stats import poisson

def calculate_lambda(team_xwOBA, opp_xFIP, park_factor, weather_factor, 
                     bullpen_adj, is_home=True):
    """計算預期得分 Lambda"""
    league_avg_runs = 4.5  # 聯盟平均
    league_avg_xwoba = 0.320  # 聯盟平均 xwOBA
    league_avg_xfip = 4.00  # 聯盟平均 xFIP
    
    attack_adjust = team_xwOBA / league_avg_xwoba
    pitcher_adjust = league_avg_xfip / opp_xFIP
    
    base_lambda = league_avg_runs * attack_adjust * pitcher_adjust
    base_lambda *= park_factor * weather_factor
    base_lambda += bullpen_adj
    
    if is_home:
        base_lambda += 0.10  # 主場優勢
    
    return max(base_lambda, 1.0)  # 最低 1 分

def predict_game(lambda_home, lambda_away):
    """Poisson 分佈預測"""
    win_prob_home = 0
    for i in range(15):
        for j in range(i):
            win_prob_home += poisson.pmf(i, lambda_home) * poisson.pmf(j, lambda_away)
    
    total_lambda = lambda_home + lambda_away
    over_8_5 = 1 - poisson.cdf(8, total_lambda)
    
    return {
        'home_win_prob': round(win_prob_home, 3),
        'away_win_prob': round(1 - win_prob_home, 3),
        'fair_home_odds': round(1/win_prob_home, 2),
        'over_8.5_prob': round(over_8_5, 3),
        'expected_total': round(total_lambda, 2)
    }

# 使用範例
lam_home = calculate_lambda(
    team_xwOBA=0.340,   # 主隊攻擊強
    opp_xFIP=3.60,       # 客隊先發弱
    park_factor=1.03,    # 場地略有利
    weather_factor=1.02, # 天氣略有利
    bullpen_adj=-0.05,   # 牛棚略強
    is_home=True
)

lam_away = calculate_lambda(
    team_xwOBA=0.310,   # 客隊攻擊一般
    opp_xFIP=4.10,      # 主隊先發強
    park_factor=1.03,
    weather_factor=1.02,
    bullpen_adj=0.05,
    is_home=False
)

result = predict_game(lam_home, lam_away)
print(result)
# 輸出: {'home_win_prob': 0.587, 'away_win_prob': 0.343,
#        'fair_home_odds': 1.70, 'over_8.5_prob': 0.412,
#        'expected_total': 8.3}

八、關鍵提醒與最佳實踐

⚠️ 避坑指南

  1. 不要直接用 ERA 和打擊率 — 它們受運氣影響太大，預測力遠低於 xFIP/SIERA/xwOBA
  2. 左右投配對比整體數據更重要 — 同一個打者 vs 左投 vs 右投可能差 50+ wRC+
  3. 牛棚疲勞是隱形殺手 — 後援投手連投 2 天以上，λ 應扣 0.1–0.3 分
  4. 場地因子差異極大 — Coors Field（+15% 得分）vs Oracle Park（−12% 得分）
  5. BvP（打者對投手）歷史樣本太小 — 不要過度依賴，用技能指標取代

✅ 最佳實踐

  1. 優先使用技能型指標：xFIP、SIERA、xwOBA 勝於 ERA、打擊率
  2. 近 30 天數據權重大於整季數據：反映真實當前狀態
  3. 場地 + 天氣組合調整：不要單獨使用其中一個
  4. Lineup 確認後再計算：主力缺陣會顯著影響 λ
  5. 與市場比較：公平賠率 > 市場賠率 → 正期望值

📊 快速決策檢查清單

✅ 單場勝負預測：

  • □ 取得先發投手 xFIP/SIERA 和 K-BB%
  • □ 取得雙方 xwOBA（分左右投配對）
  • □ 計算 λ_home 和 λ_away
  • □ 套用 Poisson 分佈
  • □ 比較公平賠率與市場賠率

✅ 單場比分預測：

  • □ 加入場地因子 × 天氣因子
  • □ 加入牛棚疲勞調整
  • □ 計算總預期得分
  • □ 套用 Poisson CDF 計算大小分概率

為什麼不能直接用 ERA 比較不同角色？

Bill James 在 1977 年的 SABR 研究已經證明：同質量的投手，救援角色的 ERA 天生比先發低 0.25–0.50 分。

三大原因

  1. 已有人出局 — 救援投手通常帶著 1–2 出局上場，失分機會更少
  2. 只面對打者一次 — 先發投手每場面對打序 2–3 次，越後面越吃虧
  3. 全力投球 — 救援投手知道只投 1 局，可以每球都拚 100%

多功能投手（Swingman / Two-Way）的評估方法

這類投手同時承担先發和中繼任務，評估時需要分場景加權：

方法：分離式評估（Split-Evaluation）

步驟 1：將該投手的數據按「角色」拆分 - 先發場次：IP ≥ 4 的出場 - 救援場次：IP < 4 的出場 步驟 2：分別計算各角色的技能指標 - 先發模式：用 FIP / SIERA / xFIP - 救援模式：用 FIP / SIERA / xFIP（同樣適用，因為這些指標不受角色影響） 步驟 3：根據本季實際使用比例加權 綜合評分 = w_SP × SP_SIERA + w_RP × RP_SIERA 其中 w_SP = 先發局數 / 總局數 w_RP = 救援局數 / 總局數

範例：假設投手 A

角色場次局數SIERA權重
先發15 場85 IP3.6085/120 = 0.71
救援20 場35 IP3.2035/120 = 0.29
綜合 SIERA = 0.71 × 3.60 + 0.29 × 3.20 = 2.56 + 1.02 = 3.58

預測模型中如何處理多功能投手？

當多功能投手是「先發」時：

λ = 基礎 × 攻擊調整 × 先發調整 × 場地 × 天氣 + 牛棚調整 + HFA 其中「先發調整」需要特別注意： - 如果該投手本季 70% 以上是先發 → 用全年 SIERA - 如果本季 70% 以上是救援 → 用救援 SIERA × 0.9（先發通常稍弱） - 如果比例接近 → 用加權綜合 SIERA

當多功能投手是「救援」時：

牛棚調整 += (救援 SIERA - 聯盟平均牛棚 xFIP) × 0.1 注意： - 如果該投手主要是先發，突然被拿去救援 → 他的球速可能更高 - 先發轉救援通常短期表現更好（球速提升） - 但長期可能因缺乏先發節奏而衰退

當無法確認角色時（Opener 戰術）：

如果開路者是救援投手： - 前 1–2 局：用救援 SIERA（通常較低） - 第 3 局起：換上 bulk pitcher，用其先發 SIERA - 總 λ = λ_opener(1–2局) + λ_bulk(剩餘局數)

實用決策樹

🔍 評估一名多功能投手 Q1: 本季先發和救援的比例？ ├── SP ≥ 70% → 用全年 SIERA/xFIP ├── RP ≥ 70% → 用救援 SIERA/xFIP └── 50/50 或頻繁切換 → 進入 Q2 Q2: 兩角色的 SIERA 差距？ ├── 差距 < 0.30 → 用全年綜合 SIERA（最簡單） ├── 差距 0.30–0.80 → 加權計算（按局數比例） └── 差距 > 0.80 → 分場景評估（先發 vs 救援用不同 λ） Q3: 該投手最近的角色是什麼？ ├── 最近 10 場都是先發 → 偏向先發 SIERA ├── 最近 10 場都是救援 → 偏向救援 SIERA └── 角色混雜 → 用最近 30 天加權 Q4: 球速有無顯著差異？ ├── 救援球速 > 先發 1.5mph+ → 救援模式可能更強 ├── 球速相近 → 角色間無實質差異 └── 先發球速 > 救援 → 可能缺乏救援強度

Python 實作：多功能投手評估器

def evaluate_multi_role_pitcher(pitcher_data):
    """
    評估多功能投手的綜合實力
    
    pitcher_data = {
        'starter_games': 15,
        'starter_ip': 85,
        'starter_siera': 3.60,
        'reliever_games': 20,
        'reliever_ip': 35,
        'reliever_siera': 3.20,
        'starter_k_bb_pct': 0.24,
        'reliever_k_bb_pct': 0.27,
        'starter_stuff_plus': 105,
        'reliever_stuff_plus': 112,
    }
    """
    total_ip = pitcher_data['starter_ip'] + pitcher_data['reliever_ip']
    
    # 計算加權 SIERA
    w_sp = pitcher_data['starter_ip'] / total_ip
    w_rp = pitcher_data['reliever_ip'] / total_ip
    weighted_siera = w_sp * pitcher_data['starter_siera'] + w_rp * pitcher_data['reliever_siera']
    
    # 計算加權 K-BB%
    weighted_k_bb = w_sp * pitcher_data['starter_k_bb_pct'] + w_rp * pitcher_data['reliever_k_bb_pct']
    
    # 判斷角色一致性
    ip_ratio = pitcher_data['starter_ip'] / total_ip
    
    if ip_ratio >= 0.7:
        role = "primarily_starter"
        recommended_siera = pitcher_data['starter_siera']
    elif ip_ratio <= 0.3:
        role = "primarily_reliever"
        recommended_siera = pitcher_data['reliever_siera']
    else:
        role = "swingman"
        recommended_siera = weighted_siera
    
    # 評估角色適應力
    siera_gap = abs(pitcher_data['starter_siera'] - pitcher_data['reliever_siera'])
    if siera_gap > 0.8:
        adaptability = "high_variance"
    elif siera_gap > 0.3:
        adaptability = "moderate_variance"
    else:
        adaptability = "consistent"
    
    return {
        'weighted_siera': round(weighted_siera, 3),
        'recommended_siera': round(recommended_siera, 3),
        'role': role,
        'adaptability': adaptability,
        'siera_gap': round(siera_gap, 3),
        'k_bb_pct': round(weighted_k_bb, 4),
        'starter_weight': round(w_sp, 2),
        'reliever_weight': round(w_rp, 2),
    }

# 範例
result = evaluate_multi_role_pitcher({
    'starter_games': 15, 'starter_ip': 85, 'starter_siera': 3.60,
    'reliever_games': 20, 'reliever_ip': 35, 'reliever_siera': 3.20,
    'starter_k_bb_pct': 0.24, 'reliever_k_bb_pct': 0.27,
    'starter_stuff_plus': 105, 'reliever_stuff_plus': 112,
})
print(result)

關鍵提醒

  1. 不要用 ERA 直接比較不同角色的投手 — 救援 ERA 天生較低
  2. SIERA/FIP/xFIP 不受角色影響 — 是跨角色比較的黃金標準
  3. K-BB% 是最穩定的技能指標 — 無論先發還是救援都可靠
  4. Stuff+（Statcast）完全不受角色影響 — 衡量純粹的投球品質
  5. 樣本量很重要 — SIERA 在 <40 IP 時不穩定，救援數據需擴大樣本
  6. 近期狀態優先 — 最近 30 天的數據比整季數據更有預測力
  7. 角色轉換期間謹慎 — 先發轉救援或反之，短期數據可能有偏差
